Learning to Use Git

It was an ardous task. I seemed very lost. Writing git commands and remembering them is not easy. Of course, with practice it becomes perfect. Our task next few days is to push things on to git by making changes to files and using commands for add, commit and push. Then merge those files and pull from the master to create a new branch. Then continue making changes and add, commit and push.

Phew! By the end of week 1, I knew what git was and felt, I conquered the world.

Build and Deploy is under Design Arch, Development and Deployment. Maintenance under Deployment and Maintenance. We do testing right from Define Requirement stage to Deployment stage. Testing stage overlaps these stage with design, build and deploy and maintain being part of Test. While building a portfolio website keep in mind at the above processes. Define your stakeholder profile and their personas. Define a problem statement: what are we doing? and a task statement: how will we do it? Framing the problem and have a at least five solutions that will be refined to get to one good solution. This should have an ultimate impact.We had short day on Tuesday, where there was a Demo Day of Cohort 21 and 5. I interviewed 4 people and got their solutions for challenges in everyday life. On Wednesday, we had given a task of creating a website for the persona we talked about Monday during our UX Design class. I chose Mandrake the Magician, who I thought was cool but we voted to pick Ryan’s persona Angelica. The rest of the week went discussing on how to build a website that sells monitoring system made by Mega Corp, where Angelica’s mom (Charlotte Pickles) works. We researched, wireframed, created a logo, and built a website using html, css and js.
